= Julia the Viper — Status

_Version is tracked in `Cargo.toml` and `CHANGELOG.adoc`._

*Stage*: Alpha (active research)
*Assessment*: B- (solid core, proofs now compiling, verification chain still incomplete)

Julia the Viper is an experimental language with a fully implemented Rust core,
a substantial test suite, and formally verified proof artefacts that now compile
cleanly. The verification chain from proofs to executable is still incomplete.

---

== Implemented Core

- Parser and interpreter for core data/control model (`crates/jtv-core`)
- Addition-only arithmetic across 7 number systems (int, float, rational, complex, hex, binary, symbolic)
- Enforced purity model (Total ≤ Pure ≤ Impure) with type-level purity lattice
- Harvard architecture separation (Data vs Control) enforced at grammar level
- Reversibility C1 + C2: `execute_inverse`, `reversible { } -> tok`, `reverse tok`, `abandon tok`
- WASM backend (591 LOC; `list_coproc_decls`, `register_coproc_impl` for extern-coproc bridge)
- LSP server (`crates/jtv-lsp`, tower-lsp integration)
- Coprocessor extern-coproc grammar, `CoprocNamespace`, `ExternCoprocNotYetLowered` phase boundary
- `ReversalToken(u64)` opaque-ID linear token store with linearity enforcement
- Echo type system: Phase 2 design complete (effect dimension, retained-loss lineage, 3-way Safe/Neutral/Breaking)

---

== Evidence

- 531 automated tests across unit, property, fuzz, contract, and conformance suites
- Property-based testing covering all 7 number systems
- Security and boundary tests for Harvard separation
- Contract tests for invariants (addition-only, purity, reversibility, token linearity)
- Fuzz targets for parser safety (`fuzz/fuzz_targets/fuzz_parser.rs`, `fuzz_main.rs`)
- Benchmark suites (not yet tracked in CI)

=== Echo Type System (`jtv_proofs/JtvEcho.lean`)

Phase 2 Echo mechanisation (Lean 4):

- `EchoClass` inductive: `Safe`, `Neutral`, `Breaking`
- Retained-loss lineage fibre: `Echo f y := Σ x, f x = y`
- `EchoSafe` / `EchoNeutral` / `EchoBreaking` predicates
- `@echo(safe|neutral|breaking)` annotation inference rules
- 3-way numeric conversion: exact→Safe, lossy+residue→Neutral, lossy-no-residue→Breaking

== Known Gaps

- Verification chain incomplete: proofs validate models but are not wired to the executable interpreter
- Reversibility round-trip: no full `(forward ∘ reverse = id)` test at the integration level
- `typing_coercion` theorem weaker than documented — follow-up ADR needed
- Coprocessor implementation gated on PataCL Phase 1
- No mutation testing
- CI limited to Linux
- FFI (Zig emission) not yet implemented
- WASM not fully validated end-to-end
